# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

`askaide-docs/` is the **public documentation website** for AskAide AI, built with [Docusaurus](https://docusaurus.io/) 3 (React 19). It is a **static site** — no application logic, and no access to the product, backend, or database. It documents the product implemented by the sibling repos; nothing here runs in production beyond serving static HTML.

- Site title: `AskAide AI Docs`
- Served at `https://AskAide-AI.github.io` under base URL `/docs/`
- Deploys to GitHub Pages (`gh-pages` branch) via `docusaurus deploy`
- Blog is disabled; docs are served at the site root (`routeBasePath: '/'`)

## Commands

This repo uses **npm** (lockfile is `package-lock.json`; there is no `yarn.lock`). The README's `yarn` examples are leftover Docusaurus template text — use npm.

```bash
npm install     # Install dependencies
npm start       # Local dev server with live reload (docusaurus start)
npm run build   # Generate the static site into build/
npm run serve   # Serve the built site locally (docusaurus serve)
```

Other scripts: `npm run clear` (clear cache), `npm run deploy` (publish to GitHub Pages), `npm run swizzle` (eject/override theme components).

## Layout

| Path | Purpose |
|------|---------|
| `docs/` | Markdown/MDX documentation content — the actual pages. Top-level sections: `frontend/`, `backend/`, `ai-service/`, `shared-contracts/`, `reference/`, `learning-resources/`, plus `intro.md` |
| `src/` | Custom theme code — `pages/` (React pages, e.g. the homepage), `components/` (e.g. `HomepageFeatures`), `css/custom.css` |
| `static/` | Assets served as-is (`img/`, favicon, logo) |
| `docusaurus.config.js` | Site config — title, tagline, URL/baseUrl, navbar, footer, deploy settings |
| `sidebars.js` | Documentation sidebar structure (maps doc IDs into the nav tree) |
| `build/` | Generated output — **never hand-edit**; regenerated by `npm run build` |

## Guardrails

1. **This site is public.** Never add pricing, sales scripts, funding strategy, credentials, internal audits, or any `../askaide-internal/` content here. Public/private separation is deliberate.
2. **Don't hand-edit `build/`** — it is generated. Edit sources in `docs/` / `src/` and rebuild.
3. Keep content in sync with actual product behavior; stale docs mislead users and hurt SEO.

## Cross-Repo Context

Sibling repos live alongside this one under the monorepo checkout: `../frontend/`, `../Backend/`, `../ai-service/`, `../shared-contracts/`, and the private `../askaide-internal/`. This site documents the product those repos implement — when a user-facing feature changes there, the relevant page here may need updating.
